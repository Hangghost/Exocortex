## Agent 互動原則 (Agentic Principles)

- **自主性優先**：不要把 Agent 當作簡單的 API 或推斷引擎。在下達任務時，應提供目標和上下文，允許並鼓勵 Agent 自行呼叫工具（如 `bash`, `read`, `grep`）來獲取所需資料。
- **減少預處理**：除非資料獲取極其昂貴或需要特殊許可權，否則應讓 Agent 自己去獲取資料，而不是在 Prompt 中喂入大量預處理好的 context。這樣能利用 Agent 的動態決策能力處理 Corner Cases。
- **深度調查邏輯**：當 Agent 發現資訊缺失時（例如找不到日誌路徑），應引導其"向下鑽取"。如果 Crontab 裡沒有寫日誌重定向，Agent 應主動檢查指令碼原始碼是否含有內部日誌邏輯。
- **結果確定性 vs 過程確定性**：關注任務的最終交付質量，而非死守固定的執行步驟。給予 Agent 實現目標的自由度。
- **單一責任原則 (SRP-A)**：多段式任務（如 Observer-Reflector 流程）中，「冪等性判斷」和「執行操作」應分離。判斷決策的 Agent 應獨立運行確認無誤後，再觸發執行 Agent。避免在同一 Agent 內同時承擔兩種認知責任，以防邊界情況下跳過判斷直接執行。
