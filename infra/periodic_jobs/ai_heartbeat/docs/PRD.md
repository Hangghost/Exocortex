# AI Heartbeat: 漸進式披露記憶系統產品需求文件 (PRD)

## 1. 產品概述

### 1.1 願景
構建一個**Agentic 驅動的、全域性統一但按需披露的觀測記憶系統**。徹底擺脫由外部指令碼“拼湊 Prompt 並餵給 AI”的低階模式，轉而讓 AI 引擎（OpenCode-Builder）在接收到簡單的“路徑與目標”後，自主探索檔案系統、分配子任務並提純觀測結果。系統遵循 **Progressive Disclosure** 理念：記憶池是全域性的，但 Agent 接收到的上下文始終保持稀疏（Sparse）和高密度（High Density）。

### 1.2 核心價值主張
- **Agentic 自主探索**: 指令碼只負責觸發任務和提供線索（檔案路徑），AI 負責閱讀、過濾（如排除僅格式變動的 Blog）和總結。
- **漸進式披露 (Progressive Disclosure)**: 預設不載入詳細記憶，僅由 Agent 根據當前任務邏輯主動檢索相關的 L1/L2 觀測點。
- **全域性分層架構**: 
  - **L3**: 全域性硬性約束（存放在 `rules/`，全域性被動載入）。
  - **L1/L2**: 動態觀測日誌（存放在全域性記憶池，Agent 主動檢索）。
- **抗噪設計**: 利用 AI 的語義理解能力識別真正的“新內容”。例如，針對 300+ 篇 Blog 的格式變動，AI 應透過檢查後設資料（Metadata）中的建立日期來識別真正的新文章。

### 1.3 目標使用者
- **OpenCode-Builder**: 作為記憶的生產者和核心消費者。
- **開發者**: 僅作為系統邊界的定義者和記憶日誌的最終審計者。

---

## 2. 核心設計原則 (The Agentic Way)

### 2.1 拒絕 Push 模式，擁抱 Pull 模式
傳統的系統試圖把所有 Context “推送”給模型。本系統要求 Agent 具備“拉取”意識。指令碼告訴 Agent：“這些檔案變了，去把有價值的 lessons 學回來”，Agent 應該自己決定讀什麼、讀多少。

### 2.2 記憶稀疏性假設 (Sparse Context Assumption)
我們假設：對於任何給定任務，真正相關的記憶是極少數的。因此，全域性記憶池（OBSERVATIONS.md）允許不斷增長，但 Agent 必須能夠透過標籤（Tags）或關鍵字進行高效的區域性載入。

### 2.3 零摩擦資產化
記憶日誌採用純文字追加模式。它不僅是 Agent 的執行狀態機，也是開發者的知識資產。

## 3. 功能需求：三層分級體系

### 3.1 L3: 全域性約束與哲學 (Global Constraints)
- **內容**: 存放於 `rules/SOUL.md` 和 `rules/USER.md`。
- **硬性約束**: 必須包含語言風格約束（不準用大詞、不準用營銷詞、不準用引號、儘量避免 "not" 負向句式）、應對策略等。
- **載入方式**: Session 啟動時被動全域性載入。

### 3.2 L1: 每日觀測與心跳 (Daily Observation)
- **內容**: 過去 24 小時的關鍵事件、技術決策、真實的錯誤修復經驗。
- **打標格式**: `🔴 High (方法論/約束)`、`🟡 Medium (專案狀態/決策)`、`🟢 Low (任務流水)`。
- **產生方式**: 指令碼僅提供 `find` 命令找出的檔案路徑集合，交給 OpenCode-Builder。Agent 自主處理（包括呼叫 Sub-agent 讀檔案、檢查 Metadata）。

### 3.3 L2: 記憶蒸餾與反思 (Weekly Reflection)
- **職責**: 垃圾回收。
- **邏輯**: 每週執行，AI 自主讀取 L1 記憶池，刪除過期的 🟢，合併同主題的 🟡，將共性經驗晉升為 🔴。

---

## 4. 關鍵業務流 (User Story)

### 4.1 智慧體自發的心跳任務
1. **觸發**: 系統 Cron Job 觸發指令碼。
2. **輸入**: 指令碼執行 `find -mtime -1`，獲得一個長路徑列表（可能包含 300+ 篇變動的 Blog）。
3. **分配**: 指令碼啟動一個 OpenCode-Builder Session。
4. **指令**: “這是過去 24 小時變動的檔案列表。你的目標是生成觀測記錄。注意：對於 blog/ 目錄下的文章，請檢查其 Metadata 中的 Date 欄位，僅處理真正今天創作的內容。如果是格式重排，請忽略。”
5. **執行**: Agent 看到任務後，自主啟動 sub-agents (librarian/explore) 分頭讀取檔案，最後彙總輸出。
6. **產出**: 結果 Append 到全域性 `memory/OBSERVATIONS.md`。
